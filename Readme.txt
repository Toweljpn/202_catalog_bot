# ✅ プロジェクト全体サマリー（引き継ぎ用）

## ■ プロジェクトの目的

自社が持っている **製品の説明文データ** をもとに、
**ユーザーの質問に回答するチャットボット**を Web で提供したい。

* ECサイト内に組み込みたい
* まずは **無料で動くプロトタイプ** を作る
* 特に制約はないが、無料で始めたい
* 用語集ではなく *製品説明文* が元データ（重要：途中で訂正）

---

## ■ 元データの形式

CSVで提供され、列は以下：

1. 管理番号
2. 作家
3. 間口
4. 奥行
5. 高さ
6. タイトル
7. 価格
8. 本文（←製品説明文：チャット生成の主素材）

これを **検索しやすい JSONL に変換して利用したい**。

---

## ■ 現在選択しているアーキテクチャ案

複数案を提示した中で、**案A（Vercel + JSONL ローカル検索）**で進めることを選択。

### 【案A：無料・最小構成のプロトタイプ】

* **フロント（UI）**：Vercel（無料）でホスティング
* **バックエンド**：Vercel Functions（サーバーレス API）
* **データ**：JSONLファイルとして Vercel に配置して読み込み
* **AI推論**：OpenAI API をバックエンドから呼び出す

理由：

* サーバー不要・無料で実装できる
* 小〜中規模データの検索に十分
* デプロイが簡単（GitHub連携）
* プロトタイプなら最速

---

## ■ Vercel Functions の理解補助

ユーザーは経験がないため、
「**サーバー不要でクラウド上にある小さなキッチンのような API 実行環境**」と説明済み。

* `/api/xxx.js` を追加するだけで API ができる
* OpenAI API などの処理をここに書く
* 無料プランでも動作可能
* サーバー管理ナシでプロトタイプに最適

---

## ■ データ量に関する質問への回答（目安）

Vercel Functions で JSONL 検索する場合：

* 数千～1万行程度なら問題なし
* 1行あたりは〜2KB程度が目安（説明文なら普通は収まる）
* 合計数MB〜10MB前後までなら現実的
* それ以上の場合は Pinecone / Supabase 等に切り替え検討

---

## ■ 追加で依頼されたもの

1. **CSV → JSONL 変換ツール（Python）**の作成
   → コード提示済み、使い方も説明済み（ユーザーがPythonを使えるため）

2. **Vercel Functions をサルでもわかるように**
   → 初学者向け超噛み砕き解説を提供済み

## ■ 補足：チャットボット回答の全体フロー（要点）

```
ユーザー質問
        ↓
Vercel Functions
        ↓
JSONL（説明文）から関連商品を検索
        ↓
OpenAI APIにプロンプト（検索結果 + 質問）を渡して回答生成
        ↓
ユーザーに返答
```
